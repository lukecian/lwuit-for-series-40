Introduction

The Lightweight User Interface Toolkit (LWUIT) is a widget library that you can bundle with your application. Lightweight refers to the library's small footprint and moderate device requirements. With LWUIT, resource-constrained devices can have a rich look and feel. Inspired by Swing, the toolkit implements a peer-less graphics and UI model in a compact package.

LWUIT is an open source project. To join the community, visit http://lwuit.java.net.

Release Highlights

This release has the following highlights, as summarized in this section. For a list of major changes see "Major Changes in This Release", and for a full list of changes, see "All Changes in This Release".

  • GUI builder - A tool for designing screen layouts by drag and drop Components on a palette, plus an IDE integration which allows developers to hook an action to the UI.

  • IO Framework - Allows portable access to network, file system, storage, web services and a json parser.

  • New Ports - Java SE port (allows testing, debugging, profiling and demonstrating the application on the desktop), and a CDC/PBP port for embedded devices.

  • HTML4 Tag Support - Provides support for most of the HTML4 tags in the HTML component.

  • XML Parsing API

  • Enhanced Blackberry support - Includes support for the torch as well as better system integration.

  • New Components - Tabs (replacing the old TabbedPane), Slider, Popup dialogs, and the Peer Component (allowing integration of native components "in place" within a LWUIT application) are new in this release.

  • Container list - Allows for variable-sized lists, grid lists and similar effects.

  • Audio playback API - Allows portable access to media.

  • Drag and drop API - Supports touch devices and tablets.

Prerequisites

In general you will use the library with an IDE or a toolkit such as the Java ME SDK. If you have a recent version of the toolkit installed, its environment will support the addition of the LWUIT library.

A minimal environment is:

  • Microsoft Windows XP or Windows Vista.

  • Java Platform, Standard Edition Software Development Kit (Java SE SDK) version 1.6 update 4 or higher. To download the SDK software, go to http://java.sun.com/javase/downloads/index.html.

  • Java ME SDK.

    To download the current SDK, go to http://java.sun.com/javame/downloads/sdk30.jsp

    In the future the Java ME SDK will be available as a NetBeans plugin. A version of LWUIT is included in the Java ME SDK plugin and it will be upgraded with the Autoupdate feature.

Installing the LWUIT Binary

The downloadable LWUIT binary includes the toolkit libraries, the Resource Editor, documentation, and sample applications. This Toolkit is able to run on CLDC1.1, MIDP2.0/CDC PBP/SE, Blackberry, Java SE. The lib directory contains the following toolkit platforms:

  • LWUIT_MIDP.jar - Standard MIDP/J2ME binary (does NOT work with Blackberry devices, works with standard MIDP 2.x/CLDC 1.1 devices)

  • MIDP_IO.jar - LWUIT4IO's MIDP version, allows IO operations such as networking, storage and filesystem actions

  • LWUIT_Desktop.jar - Works with Java SE applications, very useful for debugging/developing/demoing without a mobile environment

  • SE_IO.jar - LWUIT4IO's Java SE version, allows IO operations such as networking, storage and filesystem actions

  • NOKIA_IO.jar - Same as the MIDP_IO.jar with a couple of Nokia specific patches for APN support

  • LWUIT_Blackberry_Legacy_Pre4_7.jar - Blackberry specific version for older devices from OS 4.2 to Blackberry OS 4.7

  • LWUIT_Blackberry_4_7_OrNewerSigned.jar - Same as the version above, however requires code signing certificate from RIM to work properly

  • RIM_IO.jar - RIM specific version of LWUIT4IO, notice that in most cases the MIDP version will work just fine. The RIM version includes a couple of RIM specific enhancements for APN

  • LWUIT_CDC_PBP_UI.jar - Version of LWUIT compiled to run on top of the CDC PBP (Personal Basis Profile)

LWUIT has been ported to other platforms. See: http://www.pader-sync.com/lwuit-for-blackberry-and-android.html

Using the Library With NetBeans

Follow these steps to add LWUIT to an existing project:

  • Download and unzip the LWUIT binary.

  • In the NetBeans Project Window, right-click on a project and select Properties. Choose the Libraries and Resources category and click the Add Jar/Zip button.

  • Browse to select the appropriate platform from the lib directory, and then click Open. Click OK.

Using the Resource Editor

The Resource Editor has been redesigned and improved in many ways. In the resource_editor directory, double-click the ResourceEdit.exe file to launch the editor.
